# Story 2.3: Implement Timeline View (Execution Timeline Visualization)

## Story Statement

**As a** developer,
**I want** to see a timeline view of all executions with their start/end times, agents, and status,
**so that** I can understand execution patterns and identify bottlenecks in the agent workflow.

---

## Acceptance Criteria

1. **Timeline displays all executions from event stream:**
   - Grouped by time buckets (hourly or 30-minute intervals)
   - Shows execution bars with agent name, duration, and status (success/error)
   - Color-coded: green for success, red for error, gray for in-progress

2. **Timeline supports 24-hour view with time axis:**
   - Horizontal scroll for events beyond current viewport
   - Time scale on left (00:00 → 24:00 or current session duration)
   - X-axis shows execution bars with proper positioning based on start time
   - Y-axis shows agent lanes (one row per agent)

3. **Clicking execution shows detail view:**
   - Modal or side panel displays execution details
   - Shows: execution_id, agent, start_time, end_time, duration, status, associated events
   - Deferred drill-down interactivity to Story 2.5

4. **Responsive and performant:**
   - Handles <5000 events smoothly
   - Lazy-renders execution bars (virtualization for long timelines)
   - Responsive grid layout (mobile: 1 column, tablet: 2 columns, desktop: full width)
   - No animations (keep UI lightweight)

5. **Integrates with Metrics Cards:**
   - Timeline appears below MetricsCards in dashboard
   - Updates in real-time as new events arrive
   - Shares eventStore with MetricsCards (no data duplication)

**Data Model:**
```typescript
interface Execution {
  execution_id: string
  agent_id: string
  start_time: string        // ISO timestamp
  end_time?: string         // ISO timestamp (null if in-progress)
  duration_ms?: number      // calculated from start_time → end_time
  status: 'running' | 'success' | 'error'
  event_count: number       // events associated with execution
  error_message?: string    // if status === 'error'
}

interface TimelineBucket {
  bucket_time: string       // Rounded to hour or 30-min interval
  executions: Execution[]
  bucket_id: string         // For virtualization keys
}

interface TimelineData {
  buckets: TimelineBucket[]
  agents: string[]          // Sorted list of unique agent_ids
  earliest_time: string     // Earliest event timestamp
  latest_time: string       // Latest event timestamp
  total_executions: number
}
```
