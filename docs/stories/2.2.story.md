# Story 2.2: Implement Overview Dashboard (Metrics Cards)

## Story Statement

**As a** developer,
**I want** to see high-level metrics on the dashboard (total executions, avg duration, error rate),
**so that** I can quickly assess system health.

---

## Acceptance Criteria

1. **Dashboard displays 4 metric cards:**
   - Total Executions (count)
   - Avg Duration (seconds)
   - Error Rate (percentage)
   - Distribution by Agent (pie chart or bar chart)

2. **Metrics are computed from real-time event stream and updated every 5 seconds**
   - Uses eventStore for current session events
   - Calculates on-the-fly from event data
   - Updates automatically as new events arrive

3. **Cards show both current-day stats and 7-day trend:**
   - Sparkline trend visualization (tiny line chart)
   - Trend direction indicator (↑/↓ arrow)
   - Percentage change from 7-day average

4. **Clicking a card drills down to detail view:**
   - Filters events by agent or error type
   - Shows breakdown by agent in table
   - Shows breakdown by error type (if applicable)
   - Deferred to Story 2.5 (Drill-Down Detail View)

5. **Cards are styled and responsive:**
   - Dark theme styling with Tailwind CSS
   - Responsive grid layout (1-2 columns on tablet, 4 columns on desktop)
   - Card hover effects and state indicators
   - No animations (keep UI lightweight)

**Metrics Calculation:**
- **Total Executions:** Count of all events where event_type = 'agent.started'
- **Avg Duration:** Average of duration_ms for all completed executions (event_type = 'agent.finished')
- **Error Rate:** Count of events where event_type = 'error.occurred' / total executions × 100
- **Agent Distribution:** Grouped count by agent_id, shown as pie/bar chart

**7-Day Trend:**
- Store daily metrics in memory (or localStorage for persistence)
- Calculate average from past 7 days
- Show current vs average (% change)
- Sparkline: Tiny line chart with 7 data points

**Update Interval:**
- Refresh metrics every 5 seconds
- Use setInterval with cleanup on component unmount
- Consider performance: throttle metric recalculation if >1000 events

**State Management:**
- Metrics computed from eventStore (live)
- No separate state store needed (calculated on-demand)
- Use useMemo to avoid recalculation on every render

### Technical Constraints
- Performance: Card rendering <200ms per update
- Chart rendering: Recharts should handle <5000 events smoothly
- Memory: Keep trend data in-memory (7 days × 4 metrics = negligible)
- Browser compatibility: Chrome, Firefox, Safari (same as Story 2.1)
