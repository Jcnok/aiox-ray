# Story 2.5: Implement Drill-Down Detail View (Execution Inspector)

## Story Statement

**As a** developer,
**I want** to click on any execution and see full context (CoT, logs, ADE steps, errors),
**so that** I can understand what happened during the execution.

---

## Acceptance Criteria

1. **Detail pane displays execution metadata:**
   - Shows: execution_id, agent_id, status, duration_ms, start timestamp, end timestamp
   - Status color-coded: green (#10b981) for success, red (#ef4444) for error, gray (#6b7280) for in-progress
   - Pane opens when `uiStore.selectedExecutionId` is set (from Timeline click or FlowGraph node click)
   - Close button calls `onClose` → clears `selectedExecutionId`

2. **Chain of Thought display (if available):**
   - If CoT data exists (10% sample), display formatted milestones with timestamps
   - Each milestone shows: milestone name, timestamp, order_index
   - Milestones displayed in chronological order
   - If no CoT data: show "No Chain of Thought data available for this execution"

3. **Error events display:**
   - Show all error events linked to the execution
   - Display: error message, stack trace (if available), timestamp
   - Error section highlighted with red border/background
   - If no errors: section hidden or shows "No errors"

4. **ADE steps display (if available):**
   - Show recovery/ADE steps in a collapsible timeline format
   - Each step shows: step name, status, duration
   - If no ADE data: show "No ADE steps available"

5. **Raw event JSON for debugging:**
   - Display full raw event JSON with syntax highlighting
   - Monospace font, proper indentation
   - Scrollable container for large payloads

6. **Copy-to-clipboard buttons:**
   - Copy execution_id button
   - Copy full JSON button
   - Copy error trace button (when errors exist)
   - Visual feedback on copy (brief "Copied!" indication)

7. **Collapsible sections and responsive layout:**
   - Each section (Metadata, CoT, Errors, ADE, Raw JSON) is collapsible
   - Default: Metadata expanded, others collapsed
   - Pane takes right side of layout (slide-in panel pattern)
   - Responsive: full-width overlay on mobile (<768px), side panel on desktop
   - Min-width 400px on desktop, max-width 600px

**Drill-Down Workflow:**
[Source: docs/architecture-dashboard.md#Workflow 3]
```
1. User clicks execution bar in Timeline OR node in FlowGraph
2. DrilldownPane component receives executionId (from uiStore.selectedExecutionId)
3. Pane queries execution details from eventStore (cached)
4. Display: metadata, CoT (if available), errors, ADE steps
5. All data already loaded from historical query
Total latency: ~50ms (instant from user perspective)
```

**Data Model for Detail View:**
[Source: docs/architecture-dashboard.md#DrilldownPane]
```typescript
interface ExecutionDetail {
  execution_id: string
  agent_id: string
  status: 'success' | 'error' | 'running'
  started_at: string          // ISO timestamp from first event
  finished_at?: string        // ISO timestamp from finished event
  duration_ms?: number
  input?: Record<string, any>
  output?: Record<string, any>
  error?: {
    message: string
    stack?: string
    timestamp: string
  }
  cot_milestones?: {
    milestone: string
    timestamp: string
    order_index: number
    context?: Record<string, any>
  }[]
  ade_steps?: {
    step_name: string
    status: string
    duration_ms?: number
  }[]
  raw_events: Event[]          // All events for this execution_id
}
```

### For @dev (Implementation)
- DrilldownPane subscribes to `uiStore.selectedExecutionId`
- When `selectedExecutionId !== null`, fetch events via `eventStore.getEventsByExecutionId()`
- Transform events into `ExecutionDetail` using `executionDetailBuilder.ts` service
- Panel slides in from right side (CSS transition, Tailwind)
- Clipboard API: `navigator.clipboard.writeText()` with fallback
- JSON syntax highlighting: Simple recursive renderer with Tailwind color classes (no external lib needed)
- CollapsibleSection: Use `useState` for open/closed, CSS transition for height animation
- Performance: All data is already in eventStore (no API calls), ~50ms render target
- Responsive: Detect viewport width, overlay on mobile, side panel on desktop
- Reuse eventStore infrastructure from Story 2.1 (no duplication)
