# Story 2.4: Implement Interactive Flow Graph (Agent Call DAG Visualization)

## Story Statement

**As a** architect,
**I want** to see a directed graph of agent calls and skill invocations,
**so that** I can understand the dependency structure and bottlenecks in execution chains.

---

## Acceptance Criteria

1. **Flow graph displays agents and skills as nodes:**
   - Each unique agent_id renders as a node
   - Skill execution events render as separate nodes
   - Node size proportional to total execution duration
   - Node color indicates status: green (#10b981) for success, red (#ef4444) for error, gray (#6b7280) for mixed/in-progress

2. **Edges show execution relationships:**
   - Directed edges from parent agent to child agent/skill
   - Edge weight/thickness indicates invocation frequency
   - Edge labels show average duration between connected nodes

3. **Node hover shows summary stats:**
   - Tooltip displays: agent name, total calls, avg duration, error count, error rate %
   - Tooltip positioned near cursor, does not overflow viewport

4. **Click node for detail (deferred drill-down to Story 2.5):**
   - Clicking a node selects it (visual highlight)
   - Sets `selectedExecutionId` in uiStore for future Story 2.5 integration
   - Selection callback passed via `onSelectNode` prop

5. **Graph is interactive (pan, zoom, drag):**
   - React Flow built-in pan/zoom controls
   - Minimap for large graphs
   - Drag nodes to rearrange layout
   - Fit-to-view button to reset zoom

6. **Graph can be filtered by agent or time range:**
   - Agent filter toggle: hide/show specific agent nodes
   - Integrates with eventStore (shared data, no duplication)
   - Updates in real-time as new events arrive

7. **Responsive and performant:**
   - Max 50-100 nodes on screen (auto-group if needed)
   - Responsive: full width on desktop, scrollable on mobile/tablet
   - No animations beyond React Flow defaults (keep UI lightweight)

**Data Model:**
[Source: docs/architecture-dashboard.md#FlowGraph (Story 2.4)]
```typescript
interface FlowNodeData {
  agent_id: string
  node_type: 'agent' | 'skill'
  total_calls: number
  avg_duration_ms: number
  error_count: number
  error_rate: number           // 0-1
  total_duration_ms: number
  status: 'success' | 'error' | 'mixed'
}

interface FlowEdgeData {
  source_agent: string
  target_agent: string
  invocation_count: number
  avg_duration_ms: number
}

interface GraphData {
  nodes: Node<FlowNodeData>[]    // React Flow Node type
  edges: Edge<FlowEdgeData>[]    // React Flow Edge type
  agents: string[]               // Unique agent list for filtering
}
```

[Source: docs/architecture-dashboard.md#FlowGraph]

```
Event fields used:
- agent_id → node identification
- execution_id → relationship grouping
- event_type → node type (agent.started/finished → agent node, skill.executed → skill node)
- duration_ms → node size, edge label
- payload.parent_execution_id → edge creation (parent → child relationship)
- status (from payload) → node color
```
