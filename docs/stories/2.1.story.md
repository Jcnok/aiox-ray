# Story 2.1: Set Up Dashboard Infrastructure (React SPA + Collector Express)

## Story Statement

**As a** frontend developer,
**I want** a React SPA served by the Collector's Express server with live connection to the event stream,
**so that** I have a foundation to build visualizations on.

---

## Acceptance Criteria

1. **React App Structure Created**
   - React 18 + TypeScript project initialized with Vite build tool
   - Project directory: `squads/aiox-ray/dashboard/`
   - Build process configured: `npm run dev` and `npm run build` working
   - Responsive layout on desktop (1920px) and tablet (768px)

2. **Zustand State Stores Implemented**
   - `eventStore` with `addEvent()`, `setEvents()`, `clearEvents()`, `getEventsByExecutionId()` methods
   - `filterStore` with agent_id, startTime, endTime, errorType filters
   - `uiStore` with selectedExecutionId, isLoading, error state
   - All stores tested with unit tests

3. **Express SSE Endpoint Implemented**
   - New endpoint: `GET /events/stream` on Collector service
   - Returns Server-Sent Events (text/event-stream) with events as JSON
   - Handles SSE connections properly (headers, keep-alive, error recovery)
   - Max 100 concurrent connections supported
   - Tested with manual SSE client

4. **SSE Connection Hook Created**
   - React hook: `useEventStream()` that subscribes to `/events/stream`
   - Automatically connects on component mount
   - Adds events to eventStore in real-time
   - Handles disconnections with exponential backoff retry
   - Cleanup on unmount (connection close)

5. **Basic Dashboard Layout Implemented**
   - Header component with title and navigation
   - Sidebar component with agent filter + time picker (non-functional, UI only)
   - Main area displays raw event list (unformatted, for MVP testing)
   - Layout responsive on desktop and tablet

6. **SSE Stream → React Display Working**
   - Events from SSE stream appear in real-time on dashboard
   - Dashboard updates without page refresh
   - Events displayed as they arrive (simple list, no formatting)

7. **Dashboard Served from Collector**
   - React app builds to `dist/` directory
   - Collector serves React SPA at `/` (root path)
   - SPA 404 fallback to `index.html` for client-side routing
   - Collector serves static assets from `dist/`

## Dev Notes

### Architecture Context
[Source: docs/architecture-dashboard.md]

**Frontend Stack:**
- React 18 + TypeScript
- Zustand for state management (lightweight, fast)
- Vite for build (fast dev server, optimized production builds)
- Tailwind CSS for styling
- Recharts for charts (will be used in Story 2.2+)
- React Flow for graph visualization (will be used in Story 2.4)

**Backend Changes:**
- Collector Express server needs new SSE endpoint: `GET /events/stream`
- Express middleware stack: helmet, cors, express.json, rate limiter
- React SPA served as static files from Collector

**Component Structure:**
```
squads/aiox-ray/dashboard/src/
├── components/
│   ├── Dashboard.tsx           # Main container
│   ├── Layout.tsx              # Header + Sidebar + main area
│   └── EventList.tsx           # Raw event list display
├── stores/
│   ├── eventStore.ts           # Zustand: events[]
│   ├── filterStore.ts          # Zustand: filters
│   └── uiStore.ts              # Zustand: UI state
├── hooks/
│   └── useEventStream.ts       # SSE subscription hook
├── services/
│   └── sseClient.ts            # SSE connection management
├── App.tsx                     # Root component
└── index.tsx                   # Entry point
```

**Zustand Store Patterns:**
- `eventStore`: Immutable state with add/set/clear actions
- `filterStore`: Mutable filter state (agent_id, timestamps)
- `uiStore`: UI-only state (selectedExecution, loading)

**SSE Implementation:**
- Use EventSource API (built-in to browser)
- Connect to `/events/stream` on Collector
- Parse JSON events and add to eventStore
- Auto-reconnect with exponential backoff
- Handle connection errors gracefully

**Database Queries (handled by Collector):**
- Collector has PostgreSQL connection ready from Story 1.4
- Story 2.1 focuses on frontend infrastructure and SSE plumbing
- Historical queries deferred to Story 2.3 (Timeline)

### Technical Constraints
- Performance: Dashboard initial load <2 seconds, SSE connection <500ms
- Browser compatibility: Chrome, Firefox, Safari (latest versions)
- Responsive design breakpoints: desktop 1920px, tablet 768px
- No external authentication (MVP)

### Previous Story Insights
- Epic 1 (Stories 1.1-1.6) established solid data foundation:
  - EventEmitter captures agent lifecycle events
  - Collector service stores events in PostgreSQL
  - Data sanitization removes sensitive info
  - Performance overhead validated at 2.80% (<5% threshold)
- All infrastructure ready; Story 2.1 focuses on frontend presentation layer

### File Locations
- **Frontend code:** `squads/aiox-ray/dashboard/src/`
- **Build output:** `squads/aiox-ray/dashboard/dist/`
- **Collector public dir:** `squads/aiox-ray/collector/public/` (will copy dist here)
- **Tests:** `squads/aiox-ray/dashboard/tests/unit/` and `tests/integration/`
