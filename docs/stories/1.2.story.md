# Story 1.2: Extend Instrumentation to @qa and @architect Agents

## Story Statement

**As a** framework maintainer,
**I want** @qa and @architect agents to emit the same lifecycle events as @dev,
**so that** I have complete visibility into all core agent executions.

---

## Acceptance Criteria

1. **@qa Agent Events**
   - @qa emits `agent.started` event when activated (same schema as @dev)
   - @qa emits `agent.finished` event when completed (same schema as @dev)
   - Events are sent to same Collector endpoint with same authentication

2. **@architect Agent Events**
   - @architect emits `agent.started` event when activated
   - @architect emits `agent.finished` event when completed
   - Events follow identical schema and delivery mechanism

3. **Code Reuse**
   - No code duplication; both agents reuse EventEmitter from Story 1.1
   - Hook registration pattern is identical across all three agents
   - Single source of truth for event schema and transport logic

4. **Execution ID Traceability**
   - Each agent activation generates unique execution_id (UUID)
   - execution_id links related events from same agent run
   - Collector can correlate events by execution_id

5. **No Agent Functionality Altered**
   - @qa behavior unchanged; instrumentation is transparent
   - @architect behavior unchanged; instrumentation is transparent
   - Both agents produce identical output with/without instrumentation

6. **Measured Performance**
   - Instrumentation adds <1% overhead to each agent (same as @dev)
   - No blocking on event emission

## Dev Notes

### Architecture Context
- **Source:** docs/architecture.md#CLI Instrumentation, #Components
- **Reuse Pattern:** EventEmitter from Story 1.1 `packages/aiox-instrumentation/src/event-emitter.js`
- **Agent IDs:** "qa" and "architect" (enum in event schema)
- **Hook Points:** Same as @dev (activation and completion)

### Implementation Approach
1. Locate @qa agent definition and add EventEmitter initialization
2. Add `agent.started` hook to @qa activation
3. Add `agent.finished` hook to @qa completion
4. Repeat for @architect agent
5. Ensure same event schema and delivery as @dev
6. Verify no performance impact via benchmarking (detailed in Story 1.6)

### File Locations
- **@qa Agent Definition:** `.aiox-core/agents/qa.md` (or equivalent source)
- **@architect Agent Definition:** `.aiox-core/agents/architect.md` (or equivalent source)
- **EventEmitter (reused):** `packages/aiox-instrumentation/src/event-emitter.js`
- **Tests:** Update `packages/aiox-instrumentation/tests/event-emitter.test.js` with @qa and @architect test cases

### Testing Requirements
- Unit test: @qa emits correct event structure
- Unit test: @architect emits correct event structure
- Integration test: Both agents' events reach Collector successfully
- Performance test: Overhead validation (deferred to Story 1.6)

### Constraints
- **Dependency:** Story 1.1 must be complete (EventEmitter ready)
- **Agent Locations:** May require exploration of AIOX codebase to find exact @qa and @architect entry points
- **Backward Compatibility:** Instrumentation must not affect agents running on older AIOX versions
